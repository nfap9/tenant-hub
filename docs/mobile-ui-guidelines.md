# 移动端交互规范

这份规范用于约束 `apps/mobile` 的业务页面设计，避免页面级导航、实体卡片和表单操作混在一起。

## 层级判断

设计任何功能入口前，先判断它属于当前页面内容、原地展开、弹窗、抽屉、底部面板还是独立页面。新增层级不等于必须跳转页面；要选择最小但足够承载任务的层级。不要先把表单放到页面里再靠视觉样式补救。

### 必须新增层级

满足任意一条，就不能直接常驻展示在当前页面：

- 输入字段达到 3 个及以上。
- 需要展示可能增长的选项集合，例如角色、费用规格、套餐、房间批量模板。
- 需要用户完成多个步骤，例如选择对象、填写信息、确认结果。
- 操作有明显业务后果，例如签约、退租、删除、转移所有者、变更角色。
- 需要解释上下文或展示确认信息，否则用户容易误操作。
- 功能只服务少数使用时刻，不是页面的主要浏览内容。
- 展开后会把当前页面主要信息向下挤出首屏。
- 同一页面已经存在另一个展开表单或操作面板。

### 可以留在当前层级

同时满足以下条件，才可以直接放在当前页面或卡片内：

- 只是阅读信息、状态标识、统计摘要或导航入口。
- 操作是低风险、单击即可完成或可立即撤销。
- 输入不超过 2 个字段，且字段含义无需额外解释。
- 选项数量固定且很少，通常不超过 3 个。
- 不会改变当前页面的信息密度和浏览节奏。

### 层级与组件选择

- **原地展开**：适合列表项的轻量详情和少量操作入口。用户点哪张卡，就在那张卡展开；不要把详情放到列表底部。
- **弹窗**：适合确认、警告、二次确认、少量固定选项。弹窗内不放长表单，不承载复杂滚动。
- **底部面板**：适合移动端的短表单、筛选、排序、少量选择和快捷操作。内容应能在一屏内完成；超过一屏改抽屉或页面。
- **抽屉**：适合中等复杂度表单、可增长选项列表、需要保留背景上下文的编辑任务。抽屉可以滚动，但必须有清晰标题、关闭入口和底部提交动作。
- **独立子页面**：适合长表单、多步骤操作、高风险操作、需要完整阅读空间的任务。示例：调整成员角色、编辑组织信息。
- **详情页**：适合从列表进入后持续查看和管理某个对象，例如公寓详情、账单详情。

### 组件选择原则

- 选择能完成任务的最轻层级，但不能为了少跳转牺牲清晰度。
- 用户需要保留列表上下文时，优先原地展开、底部面板或抽屉。
- 用户需要专注完成复杂任务时，优先独立子页面。
- 高风险操作至少使用弹窗确认；如果还需要填写原因或检查影响范围，使用抽屉或子页面。
- 弹窗只做短决策，抽屉做中等任务，页面做完整流程。

### 判断流程

1. 先问：用户当前是在浏览，还是在执行一个任务？
2. 如果是浏览，只显示信息和入口。
3. 如果是任务，数输入字段、选项数量、步骤数量和风险等级。
4. 任一指标超过当前层级承载能力，就新增层级。
5. 新增层级时先选择原地展开、弹窗、底部面板或抽屉；只有任务复杂度需要时才使用页面跳转。
6. 新增层级后，返回或关闭入口必须清晰，提交动作必须在表单底部或固定操作区。

## 页面层级

- 列表页默认只展示列表、筛选、摘要和创建入口。
- 列表页不承载编辑、签约、退租、批量录入等长表单；选择列表项后最多展示摘要和操作入口。
- 列表项需要展示更多信息时，优先在被点击的卡片原地展开，避免把详情或操作面板放到列表底部造成上下文跳转。
- 详情页用于查看单个业务对象，先展示摘要信息，再展示该对象下的管理模块。
- 新建、编辑、批量录入、记录明细等表单默认不常驻显示，应由明确入口打开。
- 同一详情页存在多个轻量表单时，同一时间只展开一个表单。
- 当一个业务动作需要 3 个以上输入字段时，必须作为独立展开区、子页面或弹层处理，不能直接铺在列表或摘要区域。

## 操作放置

- 页面级导航放在页面内容顶部，不能放入实体卡片内部。
- 返回列表、返回详情、取消编辑属于页面级导航。
- 编辑当前实体、启用/停用当前条目、删除当前条目属于实体或条目级操作，可以放在对应卡片内。
- 保存、提交、确认添加属于表单级操作，应放在表单底部。
- 卡片内部不放“返回”按钮，除非该卡片本身是独立弹层或独立子页面容器。

## 表单展示

- 表单字段如果已有默认值，必须有固定标签，不能只依赖 placeholder 解释含义。
- 短字段成组展示时，每个字段都要有标签，例如“楼层数”“占地面积”“水费单价”。
- 详情页中的新增类表单通过入口按钮展开，入口按钮文案使用动作名称，例如“记录花费”“添加费用项”“批量添加”。
- 列表项的业务动作通过入口按钮打开表单，例如“编辑房间”“签约入住”；入口本身不能直接变成长表单。
- 选项集合可能增长时，不能在业务卡片中直接铺开所有选项；应使用独立选择层、子页面或弹层，例如成员角色选择。
- 表单提交成功后应自动收起，并回到详情展示状态。

## 卡片职责

- 列表卡片用于进入详情，只展示识别信息和关键摘要。
- 详情摘要卡片用于展示当前对象信息，可以放“编辑”这类对象级操作。
- 明细列表条目可以放启用/停用、选择、删除等条目级操作。
- 卡片不承担页面导航职责。

## 文案

- 按钮文案要说明动作结果，不使用含糊的“确定”“操作”。
- 返回按钮文案要说明返回目标，例如“返回公寓列表”“返回公寓详情”。
- 空状态文案说明下一步入口，例如“暂无公寓，点击新建开始维护”。
