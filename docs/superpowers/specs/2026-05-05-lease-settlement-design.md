# Lease Settlement Design

## Goal

退租必须生成一张可追踪的退租结算单，覆盖押金退还、押金扣款、手动房租退补、退租水电、其他费用、应收/应退净额，以及后续收款或退款记录。

## Rules

- 房租退补由用户手动填写，正数表示租客补交，负数表示退还租客。
- 押金默认全额退还，用户可填写押金扣款和扣款原因。
- 退租时直接填写退租水电读数，系统用租约水电单价计算水电费。
- 退租读数不能小于上次读数。
- 其他费用作为租客应补交金额。
- 净额为正时显示租客应补交，可登记收款；净额为负时显示应退租客，可登记退款；净额为 0 时显示结清。

## Data Model

新增 `LeaseSettlement`：

- 关联 `organizationId`、`leaseId`、`roomId`
- 保存退租类型、退租日期、原因
- 保存押金、押金扣款、扣款原因、房租退补
- 保存上次/退租水电读数、水电金额、其他费用
- 保存应收金额、应退金额、净额和状态

新增 `SettlementPayment`：

- 关联结算单和操作人
- `direction = RECEIVE | REFUND`
- 保存金额、方式、备注、时间

## Backend Flow

`POST /api/leases/:id/settlements` 创建退租结算单：

1. 校验租约属于当前组织且未终止。
2. 校验到期解约日期不能早于原租约结束日期。
3. 查找退租日期前最近水表、电表读数作为上次读数，若没有则按 0 处理。
4. 校验退租读数不小于上次读数。
5. 计算水电、押金退还、应收、应退、净额。
6. 在事务中创建结算单，创建退租水电读数，终止租约，释放房间。

`POST /api/leases/settlements/:id/payments` 登记结算单收款或退款：

1. 净额为正只允许 `RECEIVE`。
2. 净额为负只允许 `REFUND`。
3. 金额不能超过剩余待处理金额。
4. 收/退款达到净额绝对值后，结算单状态为 `SETTLED`。

## Mobile Flow

房间页点击退租打开退租结算表：

- 展示租约、押金、上次水电读数。
- 输入退租类型、日期、原因、押金扣款、扣款原因、房租退补、退租水电读数、其他费用。
- 前端实时展示应收、应退、净额。
- 确认后创建结算单并释放房间。

账单页后续可扩展展示退租结算单和登记收/退款。本次先在退租成功后通过接口返回结算单，保证数据闭环。
