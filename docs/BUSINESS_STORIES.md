- 版本: v2.0

## 一、物业管理模块

### 1.1 公寓（楼栋）管理

**US-101 公寓基础档案**

- 作为运营人员，我需要记录公寓的完整物业信息：
  - 基础信息：名称、地址、物业类型（住宅/商业/工业改造/城中村）、建筑年代、总层数、电梯数量
  - 产权信息：产权性质（自有/长租/托管）、产权面积、公摊比例
  - 房东合同：房东姓名、联系方式、合同起止日期、月租金、付款方式、递增规则
  - 运营配置：水电成本单价、公摊比例、提醒日设置
  - 消防与安全：消防等级、灭火器数量、逃生通道数量
- 以便全面掌握物业资产状况，支撑运营决策。

**US-102 公寓状态生命周期**

```
筹建中(PLANNING) -> 装修中(RENOVATING) -> 待开业(PREPARING) -> 运营中(ACTIVE) -> 停业整改(SUSPENDED) -> 退租关闭(CLOSED)
```

- 状态变更需记录原因和时间
- 只有运营中状态允许签约

**US-103 公寓运营支出**

- 记录公寓层面的各项支出：物业费、维修费、公共区域水电、电梯维保、消防检查、网络费等
- 支出按分类统计，支持月度/年度汇总
- 支出需关联到具体公寓，用于计算公寓级盈亏

**US-104 公寓可视化看板**

- 总房间数、已租数、空置数、维修中数量
- 本月应收、已收、欠收
- 入住率趋势图（最近12个月）
- 租金单价分布

### 1.2 房间管理

**US-201 房间档案**

- 房间号、楼层、户型（几室几厅）、朝向、面积
- 装修标准：毛坯/简装/精装/豪华装、装修日期
- 设施清单：床、衣柜、空调、热水器、洗衣机、冰箱、网络等（支持自定义）
- 房间状态及流转规则：

```
待装修(TO_RENOVATE) -> 待配置(TO_CONFIGURE) -> 空置(VACANT)
                                    ↑                        |
                                    |                        ↓
                              退租检修中(CHECKOUT_CLEANING)  <-  已预订(RESERVED) <-  维修中(MAINTENANCE)
                                    ↑
                                    |
                              已入住(OCCUPIED)
```

- 状态变更约束：
  - VACANT → RESERVED: 需关联预订记录或预付款
  - RESERVED → VACANT: 释放预订
  - VACANT → OCCUPIED: 必须有ACTIVE租约
  - OCCUPIED → MAINTENANCE: 需记录维修工单
  - OCCUPIED → CHECKOUT_CLEANING: 退租结算完成后自动流转
  - CHECKOUT_CLEANING → TO_CONFIGURE: 清洁检修完成
  - 任意状态 → VACANT: 管理员手动确认，需检查无未结事项

**US-202 房间批量管理**

- 支持按楼层/户型批量创建房间
- 批量设置房间设施模板
- 批量调整租金定价

**US-203 房间维修工单**

- 工单来源：租客报修/运营巡检/退租检修
- 工单类型：水电/门窗/墙面/家具家电/网络/管道/其他
- 工单状态：

```
待处理(PENDING) -> 已派单(DISPATCHED) -> 处理中(IN_PROGRESS) -> 待验收(AWAITING_ACCEPTANCE) -> 已完成(COMPLETED) -> 已取消(CANCELLED)
```

- 工单字段：房间、报修人、联系电话、问题描述、优先级（紧急/一般）、预约上门时间、维修人员、材料费用、人工费用、完成照片、验收备注
- 紧急工单（漏水/断电）自动短信/微信通知
- 维修费用自动计入公寓运营支出或租客账单（按责任判定）

**US-204 入住/退租检查清单**

- 入住检查：水电表底数、设施完好情况、钥匙数量、卫生状况
- 退租检查：设施损坏情况、卫生状况、需扣款项目
- 对比入住和退租状态，自动生成扣款建议
- 支持拍照留存
- 检查结果关联到租约和押金扣款

### 1.3 房东合同管理

**US-301 房东合同档案**

- 合同编号、签约日期、起止日期、付款方式（月付/季付/半年付/年付）
- 租金金额、押金金额
- 租金递增规则：固定金额/百分比、递增周期
- 免租期设置
- 合同附件（PDF扫描件）
- 合同到期前30天、7天自动提醒

**US-302 房东付款计划**

- 根据合同自动生成付款计划（应付日期、金额）
- 记录实际付款（日期、金额、凭证号）
- 逾期未付预警
- 付款计划与公寓运营支出一键关联

## 二、运营模块

### 2.1 租客档案管理

**US-401 租客基础档案**

- 姓名、手机号（唯一标识）、身份证号、身份证照片（正反面）
- 紧急联系人：姓名、关系、电话
- 工作单位、职位、收入范围（选填）
- 来源渠道：58同城/豆瓣/贝壳/老带新/中介/其他
- 信用评估：内部评分（基于历史缴费记录、退租检查）

**US-402 同住人管理**

- 一个租约可登记多名同住人
- 同住人信息：姓名、身份证号、手机号、与主租客关系
- 同住人数量限制（按户型）

**US-403 租客账户体系**

- 预付余额：提前缴纳的房租/杂费余额，可用于抵扣未来账单
- 押金余额：已缴纳押金总额
- 待付总额：所有未结清账单合计
- 净余额：预付余额 - 待付总额（正数表示预存，负数表示欠款）
- 交易流水：每一笔资金变动的时间、类型、金额、关联单据、操作人
- 账户调账：支持正负调账，需填写原因，生成审计记录

---

### 2.2 租约管理

**US-501 租约签订**

- 必填信息：房间、租客、租期起止、租金周期（月/季/年）、租金金额
- 账单日设置：每月几号出账（默认1号）
- 水电单价设置（可覆盖公寓默认值）
- 押金规则：押N付M（默认押一付一）
- 附加费用：管理费、卫生费、网络费等（名称、金额可自定义）
- 免租期设置（可选）
- 自动续约开关
- 滞纳金规则：逾期天数 × 日费率（默认万分之五）
- 签约时生成：押金账单 + 首月预付账单
- 签约后房间状态自动变为 OCCUPIED

**US-502 租约状态机**

```
草稿(DRAFT) -> 待生效(PENDING) -> 生效中(ACTIVE)
                              ↓
                         即将到期(EXPIRING_SOON) [到期前30天自动标记]
                              ↓
                         退租结算中(TERMINATING) -> 已退租(TERMINATED)
                              ↓
                         到期关闭(EXPIRED) [到期后未续约]
```

- 约束规则：
  - 同一时间一个房间只能有一个 ACTIVE/PENDING 租约
  - 租约到期后若 autoRenew=true，自动延长周期并生成新账单
  - 租约到期后若 autoRenew=false，状态变为 EXPIRED，房间状态变为 CHECKOUT_CLEANING

**US-503 租约变更**

- 支持变更：租金金额、押金金额、水电单价、附加费用、账单日
- 每次变更生成变更记录（变更前/后值、变更时间、操作人、原因）
- 变更后账单按新规则生成
- 不允许变更已终止租约

**US-504 续租管理**

- 到期前30天系统提醒运营人员联系租客
- 续租时：选择新租期、新租金（可修改），生成新的租约周期
- 续租记录关联原租约
- 支持不涨价续租/涨价续租
- 续租后原租约状态变为 RENEWED，新租约继承房间

**US-505 换房管理**

- 支持租客在租期内更换房间
- 换房流程：退租结算（原房间）→ 新签约（新房间）
- 换房时押金可转结，租金按实际入住天数计算
- 换房记录关联原租约和新租约

**US-506 退租结算**

- 退租类型：到期退租/协商退租/违约退租/强制清退
- 结算项目：
  - 租金调整：按实际入住天数多退少补
  - 水电结算：退租当日抄表
  - 押金处理：退还/抵扣（设施损坏/欠费/违约金）
  - 违约金：按租约约定计算
  - 赔偿金：设施损坏赔偿
  - 其他费用
- 结算流程：
  1. 发起退租 → 2. 退租检查（拍照）→ 3. 费用核算 → 4. 双方确认 → 5. 资金收付 → 6. 完成
- 退租完成后：
  - 结清所有未结账单（自动生成结清支付记录）
  - 租约状态变为 TERMINATED
  - 房间状态变为 CHECKOUT_CLEANING
  - 更新租客信用记录

---

### 2.3 水电表具管理

**US-601 表具档案**

- 表具编号、类型（水/电/气）、安装位置（房间/公共区域）
- 初始读数、安装日期
- 状态：在用/已拆除/已更换
- 支持父子表关系（总分表核算）

**US-602 抄表管理**

- 抄表方式：人工录入/批量导入/智能表对接（预留）
- 抄表类型：月度例行/入住抄表/退租抄表/临期抄表
- 读数校验：本期读数 < 上期读数时告警
- 用量异常检测：用量超过历史均值200%时标记"疑似异常"
- 抄表记录关联账单

**US-603 水电计费**

- 支持固定单价、阶梯单价（预留）
- 总分表差额自动公摊（按面积/按人头）
- 公共区域水电分摊到各房间

## 三、财务模块

### 3.1 账单管理

**US-701 账单生成引擎**

- 预付账单（房租+杂费）：在账单日自动生成，账单周期为下一个缴费周期
- 后付账单（水电）：在账单日生成待出账单，待抄表后完成出账
- 押金账单：签约时生成
- 结算账单：退租时生成
- 支持手动生成/补发账单

**US-702 账单生命周期**

```
草稿(DRAFT) -> 待确认(BILLING) -> 待收(UNPAID) -> 部分收款(PARTIAL_PAID) -> 已结清(PAID)
                                    ↓
                                已逾期(OVERDUE) [超过宽限期] -> 已核销(WRITTEN_OFF)
                                    ↓
                                已作废(VOID) [未收款时可作废]
```

- 账单到期前3天、逾期当天自动提醒租客

**US-703 账单子项**

- 每个账单包含多个子项：房租、水费、电费、管理费、卫生费等
- 子项独立状态：未收/已收/失败/作废
- 子项支持部分收款

**US-704 滞纳金自动计算**

- 条件：账单状态为 UNPAID/PARTIAL_PAID/OVERDUE，且超过 dueDate + graceDays
- 计算：滞纳金 = 未收金额 × 日费率 × 逾期天数
- 每日凌晨自动计算，生成滞纳金子项
- 滞纳金可减免（需审批记录）

**US-705 账单操作**

- 收款：支持一次收多笔账单，自动按账龄顺序核销（先老后新）
- 部分收款：账单状态变为 PARTIAL_PAID，继续催收剩余
- 核销：长期无法收回的坏账，经审批后核销
- 作废：未收款账单可作废（需原因）
- 拆分：部分收款后可拆分剩余金额为新账单
- 合并：同一租客的多笔账单可合并收款

---

### 3.2 收款管理

**US-801 收款登记**

- 收款方式：现金/银行转账/微信/支付宝/POS机/抵扣余额/其他
- 收款时可关联具体账单，也可作为预付款存入租客账户
- 支持一笔收款拆分到多笔账单
- 收款后自动生成出纳日记账记录

**US-802 收款核销规则**

- 默认按账龄优先：先核销最早到期的账单
- 支持指定核销某笔账单
- 超额收款存入预付余额

**US-803 退款管理**

- 退押金、退预付款、退多收款
- 退款需审批流程（大额退款需二次确认）
- 退款后更新租客账户余额

**US-804 发票管理**

- 收款后可申请开票
- 发票信息：抬头、税号、金额、内容、邮寄地址
- 发票状态：待开/已开/已寄出/已签收
- 电子发票支持（预留）

---

### 3.3 押金管理

**US-901 押金全生命周期**

```
待收(UNPAID) -> 已收(PAID) -> 部分退还(PARTIAL_REFUNDED) -> 全额退还(FULLY_REFUNDED)
                        ↓
                      已抵扣(DEDUCTED) [抵扣欠费/损坏赔偿]
```

- 押金台账：记录每一笔收取、退还、抵扣的时间、金额、原因、操作人
- 押金与租约绑定，退租时自动参与结算
- 换房时押金可转结到新租约

**US-902 押金统计**

- 当前在押总额
- 待退总额（已退租未退押）
- 各状态分布

---

### 3.4 出纳日记账

**US-1001 统一资金流水**

- 所有资金往来统一记录到出纳日记账
- 字段：日期、收支类型（收/支）、科目分类、金额、支付方式、对方（租客/房东/供应商）、关联单据（账单号/合同号）、摘要、操作人
- 支持按科目、日期、公寓、租客筛选
- 收支科目体系：
  - 收入：房租收入、押金收入、水电收入、杂费收入、违约金收入、赔偿金收入、其他收入
  - 支出：房东租金、押金退还、维修支出、物业支出、采购支出、工资支出、税费支出、其他支出

**US-1002 资金账户管理**

- 支持多账户：现金、银行账户、微信商户、支付宝商户
- 各账户余额实时统计
- 账户间转账记录
- 每日资金日报：期初余额 + 本日收入 - 本日支出 = 期末余额

---

### 3.5 财务报表

**US-1101 应收应付报表**

- 应收账款明细：按租客/房间/公寓汇总，区分已到期/未到期
- 逾期账款分析：逾期天数分布、逾期金额TOP10
- 应付账款：房东租金付款计划

**US-1102 收支报表**

- 收入统计：按科目/按月/按公寓
- 支出统计：按科目/按月/按公寓
- 收支对比：收入 - 支出 = 毛利
- 公寓级盈亏分析：各公寓的收入、成本、毛利

**US-1103 收缴率分析**

- 月度收缴率 = 本月实收 / 本月应收
- 年度收缴率趋势
- 欠费租客名单及金额
- 收缴率排名（按公寓/管家）

**US-1104 入住率分析**

- 当前入住率 = 已住房间 / 总房间
- 历史入住率趋势（12个月）
- 平均空置天数
- 续约率、退租率

## 四、系统与权限模块

### 4.1 权限管理

**US-1201 角色权限**

- 系统预置角色：
  - 超级管理员：全部权限
  - 组织所有者：组织内全部权限
  - 财务专员：账单、收款、退款、报表
  - 运营管家：公寓、房间、租约、租客、维修工单
  - 只读用户：查看所有数据，不可修改
- 支持自定义角色，按功能模块粒度授权

**US-1202 数据隔离**

- 组织级别隔离：用户只能看到自己组织的数据
- 公寓级别隔离（可选）：管家只能看到自己负责的公寓

### 4.2 审计日志

**US-1301 操作审计**

- 记录所有数据变更：操作人、时间、IP、操作类型、变更字段、旧值、新值
- 敏感操作：收款、退款、调账、退租结算、作废账单、删除记录
- 审计日志不可删除、不可修改

### 4.3 数据一致性约束

**US-1401 核心业务约束**

- C1: 同一时间一个房间只能有一个 ACTIVE 或 PENDING 租约
- C2: 账单金额 = 各子项金额之和
- C3: 租客账户净余额 = 预付余额 + 押金余额 - 待付总额
- C4: 押金已收 ≥ 已退 + 已抵扣
- C5: 账单已收 ≤ 账单总额
- C6: 租约状态变更必须按状态机规则
- C7: 房间状态变更必须关联对应业务单据
- C8: 已收款账单不可删除

**US-1402 数据校验**

- 租约日期不可重叠
- 水电读数必须递增
- 金额字段必须 ≥ 0
- 必填字段校验

## 五、业务流程图

### 5.1 新签租约流程

```
1. 选择房间（状态必须为 VACANT）
2. 录入租客信息（或选择已有租客）
3. 填写租约条款（租期、租金、押金、水电单价、附加费用）
4. 选择账单日、宽限期、是否自动续约
5. 系统校验：房间是否空闲、租客是否有未结清欠款（可选阻断）
6. 生成租约记录（状态：ACTIVE）
7. 生成押金账单 + 首月预付账单
8. 更新房间状态为 OCCUPIED
9. 通知租客（短信/微信）
```

### 5.2 月度账单生成流程

```
1. 定时任务每日凌晨执行
2. 遍历所有 ACTIVE 租约
3. 判断是否需要生成账单（账单日匹配）
4. 生成预付账单：房租 + 杂费
5. 生成后付账单：水电（待抄表后出账）
6. 更新账单队列（nextBillDate）
7. 通知租客（短信/微信）
```

### 5.3 退租结算流程

```
1. 发起退租申请（选择退租类型、日期、原因）
2. 系统锁定租约（状态变为 TERMINATING）
3. 退租检查（拍照、设施清点）
4. 录入退租水电读数
5. 系统计算结算金额：
   - 租金多退少补
   - 水电费
   - 押金退还/抵扣
   - 违约金/赔偿金
6. 生成退租结算账单
7. 结清所有未结账单
8. 资金收付（收款或退款）
9. 更新租约状态为 TERMINATED
10. 更新房间状态为 CHECKOUT_CLEANING
11. 更新租客信用记录
```

### 5.4 收款流程

```
1. 选择账单（支持多选）或输入预付款
2. 输入收款金额、方式、时间
3. 系统按账龄自动分配（或手动指定）
4. 校验：收款金额 ≤ 应收总额
5. 生成 Payment 记录
6. 更新账单状态
7. 更新租客账户余额
8. 生成出纳日记账记录
9. 生成押金台账记录（如为押金收款）
10. 支持打印收据
```

## 六、非功能性需求

### 6.1 性能

- 列表查询支持分页，默认20条/页
- 报表查询最大时间范围12个月
- 定时任务不影响正常业务响应

### 6.2 安全

- 敏感操作需二次确认
- 金额修改需填写原因
- 数据删除为软删除
- 审计日志永久保留

### 6.3 可靠性

- 定时任务支持失败重试（最多3次）
- 账单生成支持手动补发
- 数据校验在应用层和数据库层双重保障
