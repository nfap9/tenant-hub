# Tenant Hub 业务流程与 API 设计

> **设计原则**: 以业务流程为驱动，每个流程的每个节点对应一组 API 或系统行为，而非简单的资源 CRUD。
> 本文档为 API 设计的唯一权威来源，新 API 必须归属到以下某个流程的某个节点。

## 数据约束清单 (US-1401)

| 编号 | 约束                                       | 涉及流程   |
| ---- | ------------------------------------------ | ---------- |
| C1   | 同一房间同时只能有一个 ACTIVE/PENDING 租约 | B, F       |
| C2   | 账单金额 = 各子项金额之和                  | C, D       |
| C3   | 租客净余额 = 预付 + 押金 - 待付            | D, E       |
| C4   | 押金已收 ≥ 已退 + 已抵扣                   | E, D       |
| C5   | 账单已收 ≤ 账单总额                        | D          |
| C6   | 租约状态按状态机变更                       | B, E, F    |
| C7   | 房间状态变更关联业务单据                   | A, B, E, H |
| C8   | 已收款账单不可删除                         | D          |
