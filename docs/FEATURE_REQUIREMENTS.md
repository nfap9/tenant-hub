# Tenant Hub 功能需求文档（FRD）

> 本文档由 `docs/BUSINESS_STORIES.md` 逐条拆解而来。
> 每个业务故事（US-xxx）被拆分为若干功能需求（FR-xxx），覆盖前端页面、后端 API、定时任务、校验规则等实现维度。
> 后续系统功能对照检查以本文档为准。
